<section id="exponents">
<div class="container">
  <div class="tag tag-blue">Act II — The Shortcut</div>
  <h2>Exponents: one small<br>number does <span class="cb">all the work</span></h2>
  <p class="lead">Instead of writing 2 × 2 × 2 × 2 × 2, write 2⁵. That tiny "5" carries the full information. Let's explore the function f(x) = aˣ.</p>

  <div class="defbox rv">
    <div style="text-align:center;font-size:11px;color:var(--mt);letter-spacing:.1em;text-transform:uppercase;margin-bottom:6px">Definition — Exponential Function</div>
    <div class="defeq">f(x) = aˣ &nbsp; where a > 0, a ≠ 1</div>
    <div style="display:grid;grid-template-columns:1fr 1fr 1fr;gap:12px;margin-top:1.25rem;text-align:center">
      <div><div style="font-size:11px;color:var(--mt);margin-bottom:4px">BASE (a)</div><div style="font-family:var(--mono);font-size:14px;color:var(--a)">a > 0, a ≠ 1</div></div>
      <div><div style="font-size:11px;color:var(--mt);margin-bottom:4px">EXPONENT (x)</div><div style="font-family:var(--mono);font-size:14px;color:var(--a2)">any real number</div></div>
      <div><div style="font-size:11px;color:var(--mt);margin-bottom:4px">f(x)</div><div style="font-family:var(--mono);font-size:14px;color:var(--a3)">always positive</div></div>
    </div>
    <div style="margin-top:1.25rem;font-size:13px;color:var(--mt);text-align:center">
      <strong style="color:var(--t)">Key difference from polynomial:</strong> In x³, x is the variable raised to a constant. In 2ˣ, x <em>is</em> the exponent — completely different beast.
    </div>
  </div>

  <h3 style="margin-bottom:.75rem">Properties of Exponents <span style="font-size:12px;color:var(--mt);font-weight:400">(tap to see proof)</span></h3>
  <div class="pgrid rv">
    <div class="pc" onclick="this.classList.toggle('open')">
      <div class="pt">xᵐ × xⁿ = xᵐ⁺ⁿ</div><div class="pn">Product rule</div>
      <div class="pp">2³ × 2² = (2·2·2)·(2·2) = 2⁵. You're just counting total 2s: 3 + 2 = 5 ✓</div>
    </div>
    <div class="pc" onclick="this.classList.toggle('open')">
      <div class="pt">xᵐ/xⁿ = xᵐ⁻ⁿ (m > n)</div><div class="pn">Quotient rule</div>
      <div class="pp">2⁵/2² = 2·2·2·2·2 / 2·2. Two 2s cancel top and bottom → 2³. So 5−2 = 3 ✓</div>
    </div>
    <div class="pc" onclick="this.classList.toggle('open')">
      <div class="pt">xᵐ/xⁿ = 1/xⁿ⁻ᵐ (m &lt; n)</div><div class="pn">Quotient rule (m&lt;n)</div>
      <div class="pp">2²/2⁵ = 1/(2⁵⁻²) = 1/2³ = 1/8. The larger exponent wins, goes to denominator ✓</div>
    </div>
    <div class="pc" onclick="this.classList.toggle('open')">
      <div class="pt">(xᵐ)ⁿ = xᵐⁿ</div><div class="pn">Power of power</div>
      <div class="pp">(2³)² = 2³ × 2³ = 2⁶. Repeating group n times: 3×2 = 6 ✓</div>
    </div>
    <div class="pc" onclick="this.classList.toggle('open')">
      <div class="pt">x⁻ᵐ = 1/xᵐ</div><div class="pn">Negative exponent</div>
      <div class="pp">x⁰ ÷ xᵐ = x⁰⁻ᵐ = x⁻ᵐ. But x⁰=1, so x⁻ᵐ = 1/xᵐ ✓. Also: 1/x⁻ᵐ = xᵐ</div>
    </div>
    <div class="pc" onclick="this.classList.toggle('open')">
      <div class="pt">x⁰ = 1 (x ≠ 0)</div><div class="pn">Zero exponent</div>
      <div class="pp">x¹ ÷ x¹ = x¹⁻¹ = x⁰. But any nonzero ÷ itself = 1. So x⁰ = 1 ✓. Valid for all x ∈ ℝ−{0}</div>
    </div>
  </div>
  <div class="insight rv">
    <strong>Crucial observation:</strong> For a > 1, f(x) = aˣ is a <em>strictly increasing</em> function — as x increases, aˣ always increases. This means it is one-to-one: every output is reached by exactly one input. This is exactly what allows us to define its inverse — the logarithm.
  </div>
</div>

<section id="constraints">
<div class="container">
  <div class="tag tag-gold">Act V — The Rules</div>
  <h2>Not every log<br>is <span class="ca">valid</span></h2>
  <p class="lead">Three constraints, each with a mathematical reason — not arbitrary rules to memorise.</p>

  <div class="ibox rv">
    <h3>Constraint 1: N > 0 &nbsp;<span style="font-size:13px;color:var(--mt);font-weight:400">(argument must be positive)</span></h3>
    <p style="font-size:14px;color:var(--mt);margin:.75rem 0">Can 2ˣ ever equal a negative number or zero? No — for any real x, 2ˣ is always positive. So there's no x solving 2ˣ = −5 or 2ˣ = 0. Hence log₂(−5) and log₂(0) are undefined.</p>
    <div class="cgd">
      <div class="cok"><span class="cval">log₂(8)</span>8 > 0 ✓ = 3</div>
      <div class="cok"><span class="cval">log₂(0.5)</span>0.5 > 0 ✓ = −1</div>
      <div class="cbad"><span class="cval">log₂(−4)</span>−4 < 0 ✗ Undefined</div>
    </div>
    <div class="cbad" style="text-align:left;padding:1rem 1.25rem;border-radius:var(--r);margin-top:8px">
      <strong>log₂(0) = ?</strong> → asks: 2ˣ = 0. No real x satisfies this (2ˣ → 0 but never equals 0). → Undefined.
    </div>
  </div>

  <div class="ibox rv" style="margin-top:0">
    <h3>Constraint 2: a > 0 and a ≠ 1 &nbsp;<span style="font-size:13px;color:var(--mt);font-weight:400">(base restrictions)</span></h3>
    <p style="font-size:14px;color:var(--mt);margin:.75rem 0">
      <strong style="color:var(--t)">Why a ≠ 0?</strong> 0ˣ = 0 for all x>0 — not a useful function.<br>
      <strong style="color:var(--t)">Why a ≠ 1?</strong> 1ˣ = 1 always — constant function, can never reach any other value.<br>
      <strong style="color:var(--t)">Why not negative a?</strong> (−2)^(1/2) = √(−2) → complex number. We stay in real numbers.
    </p>
    <div class="cgd">
      <div class="cok"><span class="cval">log₂(x)</span>a=2>0, ≠1 ✓</div>
      <div class="cbad"><span class="cval">log₁(x)</span>a=1 ✗ 1ˣ=1 always</div>
      <div class="cbad"><span class="cval">log₋₂(x)</span>a=−2 ✗ complex</div>
    </div>
  </div>

  <div class="insight rv">
    <strong>Summary:</strong> log<sub>a</sub> N requires: &nbsp;<strong class="ca">N > 0</strong> &nbsp;and&nbsp; <strong class="ca">a > 0</strong> &nbsp;and&nbsp; <strong class="ca">a ≠ 1</strong>.
    Each constraint prevents a mathematical impossibility, not an arbitrary restriction.
  </div>
</div>
</section>
