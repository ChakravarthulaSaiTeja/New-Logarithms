Act I — Why

Why do logarithms
even exist?

Every great idea in mathematics was born to solve a real problem. Before we learn logarithms, we need to feel the problem that made them necessary.

You know multiplication

3 × 4 = 12. Simple, clean. Nothing scary here.

Repeated multiplication?

2 × 2 × 2 × 2 × 2 = 32. Fine for small numbers. Writing it out takes space.

What about 10,000 times?

Writing 2 multiplied by itself 10,000 times is literally impossible on paper. There has to be a shortcut.

Act II — The Shortcut

Exponents: one small
number does all the work

Instead of writing 2 × 2 × 2 × 2 × 2, write 2⁵. That tiny "5" carries the full information. Let's explore the function f(x) = aˣ.

Definition — Exponential Function
f(x) = aˣ   where a > 0, a ≠ 1
BASE (a)
a > 0, a ≠ 1
EXPONENT (x)
any real number
f(x)
always positive
Key difference from polynomial: In x³, x is the variable raised to a constant. In 2ˣ, x is the exponent — completely different beast.

Properties of Exponents (tap to see proof)

xᵐ × xⁿ = xᵐ⁺ⁿ
Product rule
2³ × 2² = (2·2·2)·(2·2) = 2⁵. You're just counting total 2s: 3 + 2 = 5 ✓
xᵐ/xⁿ = xᵐ⁻ⁿ (m > n)
Quotient rule
2⁵/2² = 2·2·2·2·2 / 2·2. Two 2s cancel → 2³. So 5−2=3 ✓
xᵐ/xⁿ = 1/xⁿ⁻ᵐ (m < n)
Quotient rule (m<n)
2²/2⁵ = 1/(2⁵⁻²) = 1/2³ = 1/8. Larger exponent wins ✓
(xᵐ)ⁿ = xᵐⁿ
Power of power
(2³)² = 2³ × 2³ = 2⁶. Repeating group n times: 3×2=6 ✓
x⁻ᵐ = 1/xᵐ
Negative exponent
x⁰ ÷ xᵐ = x⁻ᵐ. But x⁰=1, so x⁻ᵐ = 1/xᵐ ✓
x⁰ = 1 (x ≠ 0)
Zero exponent
x¹ ÷ x¹ = x⁰. But any nonzero ÷ itself = 1. So x⁰=1 ✓
Crucial observation: For a > 1, f(x) = aˣ is a strictly increasing function — as x increases, aˣ always increases. This means it is one-to-one: every output is reached by exactly one input. This is exactly what allows us to define its inverse — the logarithm.

Act III — A New Problem Appears

Exponents solved one thing.
But now this appeared.

Exponents tell us: "given the power, find the result." But what about the reverse?

2¹⁰⁰⁰ + 2²⁰⁰⁰ = ?
Or: what power of 2 gives you 1,000,000?
Astronomers and navigators in 1600s had to multiply massive numbers by hand. They needed a way to turn multiplication into addition.

The Forward Problem

Given power x, find result aˣ. We can do this: 2¹⁰ = 1024.

The Inverse Problem

Given result N, find power x such that aˣ = N.
This is exactly what logarithms answer.

John Napier (1614) invented logarithms to turn multiplication into addition: log(a × b) = log(a) + log(b). A genuine revolution in scientific calculation.
Act IV — The Answer

Logarithm — the
exponent finder

If 'a' is a positive real number other than 1, and aˣ = N, then we say x is the logarithm of N to the base 'a'. Written as loga N = x.

The Definition
aˣ = N  ⟺  loga N = x

Same relationship, two different ways of reading it

Exponent form (forward)
ax = N
"raise a to power x, get N"
Log form (backward)
loga(N) = x
"what power of a gives N?"

Reading it aloud
"What power must I raise 2 to, to get 32? Answer: 5."
Important note
The output of log CAN be negative (e.g. log₂(0.5) = −1), but the argument N must be > 0.

Logarithm is the inverse of the exponential. Domain and Range are swapped:
f(x) = aˣ: Domain = ℝ, Range = (0,∞)  |  f(x) = logₐ(x): Domain = (0,∞), Range = ℝ
Act V — The Rules

Not every log
is valid

Three constraints, each with a mathematical reason — not arbitrary rules to memorise.

Constraint 1: N > 0  (argument must be positive)

Can 2ˣ ever equal a negative number or zero? No. So log₂(−5) and log₂(0) are undefined.

log₂(8)8 > 0 ✓ = 3
log₂(0.5)0.5 > 0 ✓ = −1
log₂(−4)−4 < 0 ✗ Undefined
log₂(0) = ? → asks: 2ˣ = 0. No real x satisfies this → Undefined.

Constraint 2: a > 0 and a ≠ 1  (base restrictions)

Why a ≠ 0? 0ˣ = 0 — not useful.
Why a ≠ 1? 1ˣ = 1 always — can never reach any other value.
Why not negative a? (−2)^(1/2) = √(−2) → complex. We stay in real numbers.

log₂(x)a=2>0, ≠1 ✓
log₁(x)a=1 ✗ 1ˣ=1 always
log₋₂(x)a=−2 ✗ complex
Summary: loga N requires:  N > 0  and  a > 0  and  a ≠ 1.

Two types of logarithm

Common Logarithm
log₁₀(x) = log(x)
Base 10. Used in engineering, pH, Richter scale.
Natural Logarithm
logₑ(x) = ln(x)
Base e ≈ 2.71828. Used in calculus, physics.
